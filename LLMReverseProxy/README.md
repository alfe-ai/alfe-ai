# LLMReverseProxy

A reverse proxy solution that enables SSH access to local machines from a remote AWS host.

## Components

1. **Client Script** - Runs on local desktops to connect to the reverse proxy server
2. **Reverse Proxy Server** - Runs on AWS to route SSH connections to online clients

## How It Works

The LLMReverseProxy allows you to securely access local machines from a remote AWS server through SSH tunneling. It establishes a persistent WebSocket connection from your local machine (client) to the remote AWS server (proxy), enabling the proxy to forward SSH requests to your local machine.

## Running the LLMReverseProxy

### Setting up the Server (AWS)

1. Navigate to the server directory:
   ```bash
   cd LLMReverseProxy/server
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Run the server:
   ```bash
   npm start
   ```
   
   Or use the startup script:
   ```bash
   ./run-server.sh
   ```

The server will start on port 8080 by default. You can also set a custom port via the `PORT` environment variable.

### Setting up a Client (Local Machine)

1. Navigate to the client directory:
   ```bash
   cd LLMReverseProxy/client
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Configure environment variables:
   - `PROXY_HOST` - The AWS server's domain or IP address
   - `PROXY_PORT` - Port on which the server is running (default is 8080)
   - `CLIENT_ID` - Unique identifier for this client (auto-generated by default)

4. Run the client:
   ```bash
   npm start
   ```
   
   Or use the startup script:
   ```bash
   ./run-client.sh
   ```

### Environment Setup for Client

In a production setup with multiple client machines, update the `run-client.sh` script with values specific to your remote server:

```bash
export PROXY_HOST="your-aws-host.example.com"
export PROXY_PORT="8080" 
export CLIENT_ID="your-custom-client-id"
```

### Running as Services

Both client and server can be set up to start automatically on boot:

For the server:
```bash
sudo ./setup-server.sh
```

For the client:
```bash
sudo ./setup-boot.sh
```

## Features

- Seamless SSH connections from AWS to local machines
- Automatic reconnection on client restart
- Secure tunnel establishment
- Boot-time startup configuration
